Section 03

REST API Fundamentals

REpresentational State Transfer - The web communication standard

üì° What is REST API?

REST is an architectural style for designing web services. It's a standardized way for applications to communicate over the internet, like having a universal language for all web applications.

üîÑ Stateless: Each request contains all necessary information
üèóÔ∏è Client-Server: Clear separation of concerns
üíæ Cacheable: Responses can be cached for performance
üéØ Uniform Interface: Consistent way to access resources

HTTP Methods Explained

GET

Retrieve/Read Data

Fetch information without changing anything on the server. Like looking at a menu.

GET /api/users Retrieve all users
Safe Idempotent
POST

Create New Data

Add new information to the system. Like placing a new order.

POST /api/users Create a new user account
Not Safe Not Idempotent
PUT

Update/Replace Data

Replace existing information completely. Like changing your entire order.

PUT /api/users/123 Update all details for user 123
Not Safe Idempotent
PATCH

Partial Update

Modify only specific parts of existing data. Like adding extra cheese to your order.

PATCH /api/users/123 Update only email for user 123
Not Safe Not Idempotent
DELETE

Remove Data

Remove information from the system. Like canceling your order.

DELETE /api/users/123 Delete user account 123
Not Safe Idempotent
